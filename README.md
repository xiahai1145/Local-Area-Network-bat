功能列表
编号	功能名称	用途
1	快速扫描局域网存活主机	通过 Ping 和 ARP 缓存，找出在线设备，并依据 TTL 值猜测操作系统类型（Windows、Linux、macOS、Android/iOS、网络设备等）。附带实时进度条。
2	路由追踪 (tracert)	追踪数据包到达目标 IP/域名所经过的路由节点，用于诊断网络延迟和路径问题。
3	查询本机公网 IP	通过外部服务 (cip.cc) 获取您的出口公网 IPv4 地址。
4	Minecraft 服务器扫描	针对 Java 版（TCP 25565）和基岩版（UDP 19132）的专用端口探测，支持自动扫描子网、自定义范围或单 IP 检测。
5	局域网端口扫描 (通用 TCP)	自定义 IP 范围和端口列表，检测哪些 TCP 端口处于开放状态，并自动显示常见端口对应的服务名。
6	公网 IP 端口自检	自动获取您的公网 IP，并扫描预设的常用端口


无需第三方工具，基于 ping + ARP 缓存快速发现在线设备。通过分析 TTL（生存时间） 值智能识别设备类型：

TTL=128 → Windows

TTL=64 → Linux / macOS / Android / iOS

TTL=255 → 网络设备（路由器/交换机）

TTL=32 → 旧版 Windows 或嵌入式系统。

Minecraft 专用探测
同时支持 Java 版（TCP 25565） 和 基岩版（UDP 19132） 服务器端口扫描。支持自动扫描子网、自定义 IP 范围或单 IP 检测。

通用 TCP 端口扫描
自定义 IP 范围和端口列表（如 80,443,3389），快速检测开放端口。内置常用端口服务名称映射（如 80→HTTP、3306→MySQL），结果一目了然。

公网 IP 自检
自动获取你的公网出口 IP，并扫描预设关键端口（80/443/3389/8080/25565/19132），快速验证端口映射是否生效，排查路由器 NAT 环回问题。

网络诊断辅助

路由追踪（tracert -d），排查网络延迟和路径节点。

一键查询公网 IP

你可以修改脚本以适应自己的需求：

添加端口备注：编辑脚本末尾的 :get_service_name 部分，按格式添加：

batch
if "%1"=="27015" set "svc_name=Steam Game"
调整扫描速度：修改 PowerShell 命令中的 WaitOne(150 数值（单位毫秒），数值越小扫描越快，但可能漏掉延迟较高的主机。
自定义扫描范围：功能1默认扫描 /24 子网，若需扫描 /16（如 10.0.x.x），可在提示时手动输入前缀 10.0.0。
